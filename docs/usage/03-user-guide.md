# 用户侧使用指南（从“能用”到“用好”）

本文按用户侧菜单逐项说明：如何选模型、创建/管理 API Key、查看用量、使用管理令牌、配置个人设置与公告。

## 1. 仪表盘（Dashboard）

路径：**/dashboard**

用途：

1. 查看你的请求量、成本、错误率等概览（不同版本展示略有差异）。
2. 快速跳转到“使用记录/模型目录/密钥管理”等高频页面。

## 2. 模型目录（Model Catalog）

路径：**/dashboard/models**

你能做什么：

1. **搜索模型**：右上角搜索框按模型名筛选。
2. **能力过滤**：通过 Vision / Tool Use / Extended Thinking 等图标筛选。
3. **查看模型详情**：点击行可打开详情抽屉（字段通常包括：显示名、模型 ID、能力、价格、状态）。

提示：

- 模型目录展示的是“全局模型（GlobalModel）”，也就是你在 API 请求里 `model` 字段应该填写的名称。

## 3. 我的 API Keys（用于调用 Aether）

路径：**/dashboard/api-keys**

### 3.1 创建与复制 Key

1. 点击右上角“+ 创建新 API Key”。
2. 填写 Key 名称并创建。
3. 点击“复制完整密钥”按钮复制并妥善保存（建议只在可信环境保管）。

### 3.2 启用/禁用/删除

- 启用/禁用：列表行右侧“电源”按钮。
- 删除：列表行右侧“垃圾桶”按钮（不可逆）。

### 3.3 Key 能力标签（可点选的胶囊按钮）

在 Key 列表的“能力”列，你可能会看到一组可点击的能力标签（如缓存等）。

用途：

1. 给你的 Key 声明“偏好能力”（例如你希望路由优先使用支持某能力的 Key）。
2. 与管理员侧配置的模型能力/Key 能力标签联动（如果系统启用了相关策略）。

注意：能力标签是否可配置取决于系统管理员配置（你可能看到为空或不可点）。

## 4. 使用记录（Usage）

路径：**/dashboard/usage**

常用操作：

1. **选择时间段**：今天/近 7 天/近 30 天等（以 UI 为准）。
2. **筛选**：按模型、Provider、状态筛选；搜索框支持关键字过滤。
3. **自动刷新**：当存在“pending/streaming”的活跃请求时，页面会更频繁刷新状态（你也可以开启全局自动刷新开关）。

请求详情说明：

- 管理员页面会有“请求详情抽屉”，可查看更完整的出站链路与 headers 对比。
- “客户端请求头 vs 提供商请求头”的解释见：`docs/usage/request-detail-headers.md`

## 5. 管理令牌（Management Tokens，用于管理 API）

路径：**/dashboard/management-tokens**

管理令牌用于“以 Bearer Token 方式访问管理 API”（区别于用于模型调用的 API Key）。

### 5.1 创建/编辑/轮换

1. 点击右上角“+ 创建新令牌”。
2. 创建后会显示 `token_display`（脱敏），必要时可点击“重新生成令牌”进行轮换（旧令牌失效）。
3. 可编辑名称/描述、启用/禁用、删除。

### 5.2 IP 白名单（如果该令牌支持）

部分系统会允许为令牌设置 IP 白名单（仅允许特定 IP 使用该令牌）。

排障提示：

- 如果脚本使用令牌访问返回 401/403，请确认令牌启用状态、是否过期、以及请求 IP 是否被允许。

## 6. 公告（Announcements）

路径：**/dashboard/announcements**

用途：

1. 查看系统公告。
2. 支持未读计数与“点击查看详情后标记为已读”（以 UI 行为为准）。

## 7. 个人设置（Settings）

路径：**/dashboard/settings**

包含 3 类常用设置：

1. **基本信息**：用户名、邮箱、个人简介、头像 URL。
2. **修改密码**：填写旧密码与新密码。
3. **偏好设置**：
   - 主题：light/dark/system
   - 语言：简体中文/English
   - 时区：例如 `Asia/Shanghai`
   - 通知开关：邮件通知、使用提醒、公告通知（是否生效取决于管理员的邮件配置）
