    - Added support for advanced header rules in `build_safe_headers`, `PassthroughRequestBuilder`, and `build_provider_headers`.
    - Updated `ProviderEndpointCreate`, `ProviderEndpointUpdate`, and `ProviderEndpointResponse` models to accept header rules.
    - Introduced `HeadersRulesEditor` component for frontend to manage header rules.
    - Created utility functions for header rules in `header_rule_examples.py`.
    - Developed `header_rules.py` to handle the application of header rules.
    - Enhanced logging and error handling for header rule processing.
    - Updated documentation to reflect new features and usage examples.
